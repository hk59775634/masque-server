# MASQUE VPN 系统开发需求说明书（新项目）

## 1. 项目目标

建设一套可商用演进的 MASQUE VPN 系统，包含：

- Laravel 管理平台（控制面）
- MASQUE VPN Server（数据面服务端）
- Linux VPN Client（数据面客户端）
- 完整部署、监控、审计与运维体系

系统用于企业或团队统一发放 VPN 身份、下发连接配置、控制访问权限，并通过 MASQUE 协议实现加密隧道通信。

---

## 2. 建设范围

### 2.1 本期范围（Phase 1）

- Laravel 管理平台（Web + API）
- 基于 MASQUE 的认证与授权体系（证书+令牌）
- Server（支持 Linux 部署）
- Linux Client（CLI）
- 基础监控、日志、审计、告警
- 一键部署与升级方案（Dev/Staging/Prod）

### 2.2 后续范围（Phase 2+）

- Windows/macOS 客户端
- 多节点与高可用（多地域）
- 多租户与组织级隔离
- 自助订阅/计费能力（如有商业需求）

---

## 3. 角色与权限模型

### 3.1 角色定义

- 超级管理员：系统全局配置、证书根管理、节点管理
- 运维管理员：节点生命周期、发布、告警处理
- 安全管理员：策略、审计、封禁、证书吊销
- 普通用户：查看/下载本人配置，查看连接状态

### 3.2 权限要求

- 基于 RBAC（角色-权限）实现
- API 必须做鉴权与鉴权失败审计
- 高危操作（吊销、删除、轮换根证书）需二次确认与审计记录

---

## 4. 总体架构要求

## 4.1 架构分层

- 控制面：Laravel（用户、设备、策略、证书、配置、审计）
- 数据面：MASQUE Server（隧道建立、数据转发、策略执行）
- 客户端：Linux CLI（认证、建隧道、路由/DNS配置、状态上报）
- 基础设施：MySQL/PostgreSQL、Redis、对象存储（可选）、Prometheus/Grafana/Loki

### 4.2 协议与接口

- 控制面与客户端：HTTPS REST API（JSON）
- 控制面与服务端：HTTPS/gRPC（二选一，建议 gRPC）
- 数据隧道：QUIC + HTTP/3 + MASQUE

---

## 5. Laravel 管理平台需求

### 5.1 账号与认证

- 支持本地账号登录（邮箱+密码+2FA）
- 支持 OAuth2/OIDC 对接（企业可选）
- 登录风控：失败次数限制、IP 黑名单、可选验证码
- 管理员会话过期策略与强制下线

### 5.2 用户与设备管理

- 用户增删改查、状态（启用/禁用）
- 设备注册与绑定（设备名、设备指纹、公钥/证书）
- 单用户多设备限制（可配置）
- 设备吊销、临时封禁、永久封禁

### 5.3 证书与密钥管理（核心）

- 内置 CA 或对接外部 CA（优先支持内置）
- 签发客户端证书（短期证书，建议 7~30 天）
- 支持证书自动轮换与续期
- CRL/OCSP 机制（至少支持 CRL）
- 根证书轮换流程（双根过渡期）

### 5.4 MASQUE 认证流程编排

- 为每个用户/设备签发身份材料（证书或 token）
- 下发连接配置（server_addr、SNI、ALPN、策略版本、DNS）
- 支持一次性激活码（设备首次入网）
- 服务端可回调控制面进行二次鉴权（在线授权）

### 5.5 路由与策略管理

- 支持全局路由策略（全量走隧道）
- 支持分流策略（网段白名单/黑名单，后续扩展）
- 支持用户级、组级策略下发
- DNS 策略：指定 DNS、DoH/DoT 预留字段

### 5.6 审计与日志

- 审计事件：登录、设备注册、证书签发、吊销、策略变更
- 查询筛选：按用户、设备、时间、事件类型
- 日志保留策略：>=180 天（可配置）
- 审计日志不可篡改（至少实现追加写和签名校验）

### 5.7 管理 API

- OpenAPI 文档自动生成
- API 版本化：`/api/v1/...`
- 幂等性：关键写操作支持 Idempotency-Key
- 限流：用户维度/IP 维度限流

### 5.8 管理界面（Web）

- 仪表盘：在线用户数、活跃隧道、失败率、节点健康
- 用户/设备/策略/证书可视化管理页
- 一键生成 Linux 客户端配置下载
- 国际化：中文优先，英文可扩展

## 6. MASQUE Server 需求

### 6.1 基础能力

- Linux 部署运行
- 支持 QUIC/HTTP3 与 MASQUE 隧道
- 支持 TUN 设备读写与 IP 转发
- 支持多客户端并发与连接复用

### 6.2 认证与授权

- mTLS（客户端证书）优先
- 可选 token 二次认证（JWT）
- 建连时校验证书有效期、吊销状态、设备状态
- 支持动态拒绝策略（控制面实时下发）

### 6.3 路由与转发

- 支持全局模式：默认路由经隧道
- 支持客户端下发服务器保留路由（防路由黑洞）
- NAT/转发规则自动配置与自检
- MTU/MSS 参数可配置

### 6.4 策略执行

- 按用户/设备应用 ACL（目的网段、端口、协议）
- 支持会话级限速（可后续实现）
- 支持会话并发数限制

### 6.5 可观测性

- Prometheus 指标：
  - 在线连接数
  - 建连成功率/失败率
  - 握手延迟
  - 上下行流量
  - 会话时长分布
- 结构化日志（JSON）
- 健康检查端点 `/healthz`

### 6.6 高可用与扩展（预留）

- 多实例无状态化（状态放 Redis/DB）
- 支持前置负载均衡（UDP/443）
- 灰度发布与回滚机制

## 7. Linux Client 需求

### 7.1 CLI 功能

- 登录/激活：`login`、`activate`
- 连接控制：`connect`、`disconnect`、`status`
- 配置管理：`config import/export/show`
- 诊断工具：`doctor`（网络、证书、权限检查）

### 7.2 建连与认证

- 从控制面拉取配置与证书
- 支持 mTLS 建连
- 证书过期前自动续期
- 网络抖动自动重连

### 7.3 路由与 DNS

- 支持 `-route all`（全局）
- 必须自动添加服务器 IP 保留路由（防黑洞）
- 添加分段默认路由（`0.0.0.0/1` + `128.0.0.0/1`）
- 支持 `-dns` 指定 DNS，并在断开时恢复系统 DNS

### 7.4 系统兼容

- 支持 systemd-resolved 与 `/etc/resolv.conf` 两种 DNS 方式
- 对 `CAP_NET_ADMIN` 缺失给出明确错误提示
- 日志可输出到 stdout 与本地文件

## 8. 数据模型（控制面）

至少包含以下核心表：

- users（用户）
- roles / permissions / role_user
- devices（设备）
- certificates（证书元数据）
- vpn_nodes（VPN 节点）
- policies（策略）
- user_policy_bindings / device_policy_bindings
- sessions（在线会话）
- audit_logs（审计日志）
- api_tokens / refresh_tokens（如采用 token）

关键要求：

- 所有关键表包含 `created_at/updated_at`
- 高并发查询字段建立索引（user_id、device_id、status、expire_at）
- 审计日志表禁止物理删除（仅归档）

## 9. 部署需求

### 9.1 环境划分

- dev：开发联调
- staging：预发布验收
- prod：生产

### 9.2 部署方式

- Laravel：Nginx + PHP-FPM（或容器化）
- Server：systemd 服务（建议容器化可选）
- DB：MySQL/PostgreSQL（主从可后续）
- Redis：缓存与短期会话

### 9.3 CI/CD

- PR 触发：Lint + Unit Test + SAST
- 主干触发：构建镜像 + 集成测试 + 发布
- 支持一键回滚到上一个稳定版本

### 9.4 配置与密钥管理

- 严禁明文密钥入库
- 使用 `.env` + 密钥管理服务（Vault/KMS，可后续）
- 证书私钥访问最小权限

---

## 10. 安全与合规需求

- 全链路 TLS（管理面/API/控制通道）
- 密码哈希：Argon2id/Bcrypt
- 管理员强制 2FA
- 防重放、防暴力破解、基础 WAF 策略
- 审计日志合规保留与导出
- 依赖漏洞扫描（每周）

---

## 11. 测试需求

### 11.1 测试分层

- 单元测试：业务逻辑、策略计算、证书流程
- 集成测试：控制面-服务端-客户端联调
- 端到端测试：真实建隧道 + 访问验证
- 性能测试：并发连接、吞吐、延迟、重连

### 11.2 验收指标（建议）

- 单节点稳定在线连接：>= 1,000（按机器规格调优）
- 建连成功率：>= 99%
- 服务端可用性：>= 99.9%
- 关键 API P95：< 300ms
- 客户端断线重连恢复：< 10s（目标）

## 15. 风险与约束

- MASQUE 生态仍在快速演进，协议细节需持续跟进 RFC 与实现兼容
- Linux 网络栈差异（发行版、systemd、容器网络）需做兼容策略
- 证书生命周期和密钥安全管理复杂，必须优先设计
- 若需跨平台客户端，本期应先保证 Linux 质量再扩展
